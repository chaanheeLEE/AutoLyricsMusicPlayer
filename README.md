# 🎵 Auto Lyrics Player (자동 가사 플레이어)

Auto Lyrics Player는 오디오 파일을 재생하고, **AI 기반 음성 전사(Transcription) 기술**을 사용해 노래 소리로부터 실시간 싱크가 맞춘 가사를 자동으로 추출하여 보여주는 Electron 기반의 데스크톱 뮤직 플레이어 어플리케이션입니다.

---

## 🛠️ 주요 기능 및 구현 현황

### 1. 음악 재생 및 앨범 아트 추출
* MP3, WAV, FLAC, M4A, AAC 등 다양한 오디오 포맷 파일 재생 및 재생 목록(Playlist) 관리.
* **FFmpeg** 백엔드를 연동하여 오디오 파일 내에 내장된 앨범 아트를 추출하고 메인 UI에 표시.
* 셔플(Shuffle) 재생 및 다양한 재생 모드 지원.

### 2. AI 기반 자동 가사 생성 (Transcription)
* Python의 고성능 음성 인식 라이브러리인 **`faster-whisper`**를 백엔드로 구동.
* 백그라운드 프로세스에서 모델 로딩 및 전사 진행 상황(%)을 실시간 피드백.
* 분석된 데이터(시작 시간, 끝 시간, 가사 텍스트, 신뢰도)를 메인 창에 실시간 전달.

### 3. 가사 동기화 및 편리한 편집 환경
* **이진 탐색(Binary Search) 알고리즘**을 활용해 현재 오디오 재생 위치에 맞춰 가사 하이라이팅 및 자동 스크롤 기능 제공.
* 가사 싱크가 미세하게 어긋날 경우를 대비해 **오프셋(Sync Offset) 조절 기능** 구현 (단축키 또는 버튼으로 ±0.1초 / ±0.5초 조절 가능).
* **가사 편집 모드(Edit Mode)**를 지원하여 프로그램 UI 내에서 가사 텍스트를 즉시 수정 가능.
* 한 번 분석된 가사는 곡별 고유 해시 키(Cache Key)를 기준으로 로컬에 자동 캐싱되어, 동일한 곡을 재생할 때는 즉시 가사가 로드됨.

### 4. 가사 내보내기 (Export)
* 분석 및 수정한 가사 데이터를 표준 포맷인 **LRC(.lrc)** 및 **WebVTT(.vtt)** 파일 형태로 즉시 내보낼 수 있는 기능 지원.

### 5. 화면 투명 플로팅 가사 창 (Floating Lyrics Window)
* 화면에 항상 맨 위에 표시(`alwaysOnTop`)되는 별도의 **투명 가사 오버레이 창** 지원.
* 마우스로 자유롭게 창의 위치를 이동시키고 크기를 조절할 수 있으며, 이 설정(Bounds)은 로컬에 자동 저장되어 재실행 시 유지됨.
* 현재 재생 중인 가사 줄을 기준으로 이전/현재/다음 가사 3줄을 부드럽게 연동하여 표시.

### 6. 다중 가사 소스 로드 및 AI 가사 정합 파이프라인 (AI Align)
* **다중 가사 로드 우선순위**: 트랙 로드 시 아래 우선순위 파이프라인을 따라 가사를 자동으로 탐색합니다.
  1. **1순위**: `userData` 내 저장된 기존 JSON 캐시 (STT 및 AI 정합이 완료된 최상의 품질)
  2. **2순위**: 음원 파일과 동일한 디렉토리에 있는 `.lrc` 파일
  3. **3순위**: 음원 파일 내장 메타데이터의 LRC 형식 가사 (USLT, SYLT, ©lyr 등)
  4. **4순위**: 음원 파일 내장 메타데이터의 평문(Plain) 가사 (가사창에 직접 보여주지 않고 **AI Align의 공식 원문 텍스트**로만 활용)
* **Whisper STT 및 Gemini AI Align**:
  * 타임스탬프가 없는 음원은 **Analyze (Whisper STT)**를 통해 우선 기본 시간선 정보를 추출합니다.
  * 내장 평문 가사(존재 시 최우선 사용) 혹은 웹 크롤링(DuckDuckGo 기반)을 통해 얻은 공식 가사 텍스트를 **Gemini 3.1 Flash-Lite** 모델에 전달하여, Whisper의 시간선과 공식 가사를 정밀하게 일치시킵니다.
  * 시작 지점 보간, 최소 듀레이션(0.8초) 보정, 인접 동일 시간대 병합 등의 보정 메커니즘을 통해 싱크 신뢰도를 극대화합니다.
  * `autoAnalyzeMode`가 `align`으로 활성화된 경우, Whisper STT 완료 후 즉시 **AI Align이 자동 연쇄적으로 실행**되어 최종 싱크 가사를 획득합니다.

## 📂 프로젝트 폴더 및 파일 구조

프로젝트는 Electron의 권장 설계에 따라 **메인(Main) 프로세스**, **렌더러(Renderer) 프로세스**, 그리고 이들이 공통으로 참조하는 **공통(Shared) 스크립트**로 완벽히 격리 및 모듈화되어 있습니다.

```
music-player/
│
├── package.json               # Electron 의존성 및 빌드/실행 스크립트 정의
│
└── src/
    ├── main/                  # Electron 메인 프로세스 영역
    │   ├── main.js            # 메인 프로세스 시작점 (윈도우 생성, IPC 통신, FFmpeg 앨범 아트 추출 등)
    │   ├── preload.js         # Renderer와 Main 간 안전한 Context Bridge 연결 통로
    │   ├── transcribe.py      # faster-whisper 모델을 사용하여 실시간 음성 인식을 수행하는 Python 스크립트
    │   └── transcription-worker.js  # Python 프로세스 제어 및 FFmpeg 경로 찾기 등 AI 분석 스케줄러
    │
    ├── renderer/              # 사용자 화면(UI) 렌더러 프로세스 영역
    │   ├── index.html         # 메인 플레이어 UI 구조
    │   ├── app.js             # 메인 플레이어의 상태(State) 및 UI 인터랙션 관리
    │   ├── styles.css         # 플레이어 전체 스타일시트 (테마, 단축버튼, 가사 흐름 등)
    │   ├── floating.html      # 플로팅 가사 전용 창 UI
    │   └── floating.js        # 플로팅 가사 화면 렌더링 및 메인 프로세스 신호 수신
    │
    ├── shared/                # 공통 유틸리티 영역
    │   └── lyrics-core.js     # 가사 오차 보정, 싱크 검색(이진탐색), LRC/WebVTT 포맷 변환 공통 로직
    │
    └── assets/                # 앱 아이콘 및 리소스 디렉토리
```
